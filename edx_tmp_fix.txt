# Вирішення помилки "No usable temporary directory" в edX

## Проблема
Якщо ви отримуєте помилку:
```
[Errno 2] No usable temporary directory found in ['/tmp', '/var/tmp', '/usr/tmp', '/edx/app/xqwatcher/src']
```

Це означає, що Python не може знайти директорію з правами на запис для тимчасових файлів.

## Рішення

### Варіант 1: Додати код для фіксації на початку вашого скрипта

```python
# === ПОЧАТОК ФІКСУ ===
# Вставте цей код на самому початку головного файлу
import os, tempfile
tmp_dir = os.path.join(os.getcwd(), '.forced_tmp')
if not os.path.exists(tmp_dir):
    os.makedirs(tmp_dir, mode=0o777)
os.environ['TMPDIR'] = tmp_dir
os.environ['TEMP'] = tmp_dir
tempfile.tempdir = tmp_dir
# === КІНЕЦЬ ФІКСУ ===

# Ваш звичайний код далі...
import інші_модулі
```

### Варіант 2: Використати готовий патч

1. Скопіюйте файл `force_temp.py` з цього репозиторію
2. Додайте на початку вашого коду:
```python
import force_temp  # Цей імпорт налаштує тимчасові директорії
# Ваш код далі...
```

### Варіант 3: Для складніших випадків

Якщо простий варіант не працює, використовуйте детальний діагностичний скрипт:
```python
import fix_tmp_globallogic  # Патч з діагностикою
# Ваш код далі...
```

## Пояснення для інструкторів

Проблема виникає через обмеження середовища edX/xqwatcher, де стандартні тимчасові директорії не мають прав на запис. Додавання цього коду дозволяє створити і використовувати локальну тимчасову директорію для виконання тестів.
